How to build a static website with Hugo, TailWind and Github Pages

Jun 13, 2025

The Stack

If you don’t care about the reasoning behind the stack skip to Getting Started.

The purpose of the website was for a place I could host my resume, display some of my interest and share some of the knowledge I’ve gained. After all, I’ve benefited a ton from different blog post so I thought I might try and contribute.

I currently take all of my notes in markdown and didn’t want to be translating those notes to HTML myself. The two big projects for SSG (Static Site Generation) are Jekyll and Hugo. I chose Hugo simply because I knew the Go HTML template format.

As for TailWindCSS I personally enjoy getting to style and build my website all in the same file. (Though we can’t use tailwind to style the “generated” content you see here. We have to use plain old CSS for that.)

Lastly the decision for GitHub pages came down to ease of use. I originally was going to host this from my home lab with a custom Zig web server running in Kubernetes. This approach would force me to open my home lab to the Public Internet and I wouldn’t get the benefit of GitHub’s CDN (Content Delivery Network). The logical choice was the free and easy option of GitHub Pages.
